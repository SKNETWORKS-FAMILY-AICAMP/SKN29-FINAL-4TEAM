# 팀원 E - UI·시연 데이터 확인 가이드

## 목적

WPU-JAC104D 고객 문의 6종을 화면에서 시연하고, 공식 근거와 팀 설계 흐름을 구분해 표시한다.

## 필수 확인 파일

1. `data/synthetic/demo_scenarios.json`
2. `data/processed/structured/rag_verified_sample.jsonl`
3. `data/processed/metadata/model_scope.csv`
4. `data/processed/metadata/error_missing_list.csv`

## 기본 시연 시나리오

| scenario_id | 증상 | 화면 우선 처리 |
|---|---|---|
| `SYN-JAC104-001` | 무출수 | 필터·단수·호스 확인 후 상담 조건 |
| `SYN-JAC104-002` | 출수량 저하 | 동시 수전·필터·온수·수압 확인 |
| `SYN-JAC104-003` | 냉수 온도 이상 | 냉각 대기와 2시간 후 상담 조건 |
| `SYN-JAC104-004` | 누수 | 즉시 사용 중지·밸브·전원·상담 |
| `SYN-JAC104-005` | 물맛·냄새 | 미사용 기간·통수·수질 단정 금지 |
| `SYN-JAC104-006` | 순간온수 경고 | 음용 금지·즉시 상담 |

## 화면에 표시할 공식 근거

- 모델명: `WPUJAC104DWH`
- 문서명: SK매직 WPU-JAC104D/JCC104D 사용설명서
- 버전: `REV.00`
- 근거 페이지: 각 시나리오의 `page_refs`
- 출처 구분: 공식 매뉴얼 / 팀 설계 / 합성 데이터

## 위험 안내 규칙

- 누수와 순간온수 모듈 경고는 일반 팁 카드보다 사용 중지·상담 CTA를 우선한다.
- 자가 분해·직접 수리 버튼이나 문구를 제공하지 않는다.
- `requires_consultation=true`인 경우 문의 종료를 기본 선택으로 제공하지 않는다.
- 미확인 오류 코드는 의미를 추정하지 않고 고객 입력 원문과 상담 연결만 표시한다.

## 개인정보 표시

- `DEMO-*` 식별자는 합성 데이터 배지를 표시한다.
- 실제 이름·전화번호·주소처럼 보이는 값을 추가하지 않는다.
- 화면 캡처와 발표 자료에도 합성 데이터임을 표시한다.

## IAC425 확장

얼음 특화 화면은 후속 확장이다. 기본 MVP 화면에 IAC425 모델 선택이나 얼음 문의를 노출하지 않는다. 확장 시작 시 `expansion_secondary` 청크 4건으로 별도 시나리오를 만든다.

## 완료 기준

- 6개 시나리오가 JAC104D 모델과 올바른 청크를 참조함
- 공식 근거와 팀 설계 흐름이 시각적으로 구분됨
- 위험 문의 CTA와 음용·사용 중지 문구가 누락되지 않음
- IAC506 데이터가 화면에 노출되지 않음
