# WPU-IAC506 공식 자료 전처리 설계서

## 1. 목적과 범위

이 문서는 기획 단계에서 공식 자료가 RAG 입력으로 가공 가능한지 검증하기 위한 설계서다. 전체 매뉴얼·FAQ 전처리, OCR, 임베딩 생성, 벡터 DB 적재는 이번 범위에 포함하지 않는다.

이번 단계의 산출물은 다음 두 가지다.

- 전처리·메타데이터 규칙
- 검증된 매뉴얼 페이지 기반 대표 JSONL 샘플 6건

## 2. 공식 근거 우선순위

1. 제품 모델이 정확히 일치하는 공식 사용설명서
2. 제품 모델이 정확히 일치하는 공식 FAQ
3. 모델 공통 FAQ 중 사용설명서와 내용이 일치하는 항목
4. 공식 근거가 없거나 충돌하는 경우 `판단 보류·상담 필요`

WPU-IAC506 직접 언급이 없는 FAQ는 단독 근거로 사용하지 않는다. 매뉴얼과 FAQ가 충돌하면 WPU-IAC506 매뉴얼을 우선한다.

## 3. 포함·제외 규칙

### 포함

- WPU-IAC506 모델명이 확인되는 매뉴얼 구간
- 페이지와 절 제목을 추적할 수 있는 텍스트
- 표의 행·열 관계가 보존된 관리 기준
- 안전 경고, 금지 조치, 상담 전환 조건
- 매뉴얼과 내용이 일치하는 모델 공통 FAQ 후보

### 제외 또는 보류

- 다른 모델명이 명시된 FAQ
- 이미지에만 답변이 있고 이미지 원본을 검증하지 못한 FAQ
- 개별 원문 URL을 식별할 수 없는 FAQ의 단독 인용
- 가격, 계약, 방문관리 정책처럼 변경 가능성이 높고 모델 매뉴얼에 없는 정보
- 건강 효과를 단정하거나 공식 매뉴얼 범위를 벗어나는 설명
- 고객에게 분해, 부품 교체, 전기 계통 점검을 요구하는 내용
- 근거 없이 정상 사용 가능 여부나 고장 원인을 확정하는 문장

## 4. 청크 작성 규칙

- 청크 경계는 페이지 수가 아니라 하나의 의미 단위로 나눈다.
- 증상, 예상 원인, 안전한 확인, 상담 조건을 가능한 한 같은 청크에 유지한다.
- 필터 주기표는 필터별 한 행을 독립 사실로 보존하되 기준 사용량과 예외 문구를 함께 저장한다.
- 안전 경고는 일반 사용 설명과 분리하고 위험도를 별도 필드로 기록한다.
- 서로 다른 페이지의 내용을 합칠 때는 모든 페이지 번호를 남긴다.
- 원문을 임의로 보정하지 않고 요약문과 구조화 필드를 분리한다.
- 빈 50쪽은 제외한다.
- FAQ 번호는 원본 Markdown의 번호를 유지하며 새 번호를 만들지 않는다.

## 6. 안전·근거 부재 처리

- 누수, 타는 냄새, 감전·화재 가능성, 히터 경고는 일반 자가조치보다 안전 안내와 상담 연결을 우선한다.
- 사용설명서에 없는 분해·수리 절차는 생성하지 않는다.
- 공식 근거가 없는 경우 `확인 가능한 공식 근거 없음`을 표시한다.
- 공식 근거가 없는 사용 가능 여부는 `판단 보류·상담 필요`로 반환한다.
- AI는 원인을 확정하지 않고 `점검 후보`, `가능성`, `확인 필요`로 표현한다.
- 공식 자료에서 확인되지 않은 오류 코드는 의미를 추정하지 않고 고객 입력 원문을 보존한 뒤 `판단 보류·상담 필요`로 처리한다.

## 7. FAQ 적용 규칙

현재 FAQ 119건에는 WPU-IAC506 직접 언급이 없다. 따라서 이번 대표 JSONL에는 FAQ 청크를 포함하지 않는다.

향후 FAQ를 사용할 때는 다음 조건을 모두 충족해야 한다.

1. 타 모델 전용 항목이 아니다.
2. 이미지 전용 답변이거나 출처가 불명확한 항목이 아니다.
3. WPU-IAC506 매뉴얼의 사실·버튼 조작·안전 기준과 충돌하지 않는다.
4. `applicability`를 `common_unverified`로 기록한다.
5. 단독 근거가 아닌 보조 검색 후보로만 사용한다.

이 규칙은 PMD-002에 따라 최종 승인됐다. 매뉴얼과 충돌하거나 모델 적용 여부가 불분명한 FAQ는 고객 안내 근거에서 제외한다.

## 8. 대표 샘플 구성

| 샘플 | 근거 페이지 | 용도 |
|---|---:|---|
| 필터 교체 주기 | 35 | 케어 일정 |
| 출수량 저하·무출수 | 31, 43 | 점검 후보 |
| 물맛·냄새 이상 | 44 | 자가 확인·상담 조건 |
| 누수 | 5, 44 | 위험 분기 |
| 냉수 온도 이상 | 42 | 점검 후보·상담 시점 |
| 온수 이상·히터 경고 | 45 | 위험 분기·음용 중지 |

현재 대표 샘플은 온수 일반 점검과 히터 경고를 하나의 청크로 묶은 기획 검증본이다. PMD-006에 따라 실제 전처리 단계에서는 `온수 일반 점검`과 `히터 경고·비음용` 청크를 분리하고 각각의 위험도·사용 중지 범위를 독립 검증한다.

## 9. 역할 인계 기준

- 팀원 C: 공식 원본, 출처, 적용 범위, 제외 규칙과 대표 샘플 제공
- 팀원 B: 실제 청크 크기, 임베딩 모델, 검색·재순위화와 평가 방식 결정
- 팀원 D: 메타데이터 필드와 DB 저장 구조 연결
- 팀원 E: 문서명·페이지·판단 보류·상담 필요 상태의 화면 표시 확인

## 10. 검증 기준

- 모든 `document_id`가 원본 목록에 존재해야 한다.
- 모든 `page_refs` 값이 1-50 범위에 있어야 한다.
- 비연속 페이지를 사용한 청크는 `page_refs`에 실제 페이지를 빠짐없이 기록해야 한다.
- `model_exact` 청크는 WPU-IAC506 매뉴얼만 참조해야 한다.
- 위험 청크에는 상담 조건 또는 금지 조치가 있어야 한다.
- JSONL 각 줄은 독립적인 유효 JSON이어야 한다.
