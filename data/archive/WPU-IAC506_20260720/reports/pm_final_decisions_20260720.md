# WPU-IAC506 MVP PM 최종 결정

> 결정일: 2026-07-20  
> 상태: 최종 승인  
> 적용 범위: 데이터 수집·RAG·데이터 구조·화면 시연·인계 패키지

## PMD-001. MVP 지원 모델

- MVP 지원 모델을 `WPU-IAC506` 단일 모델로 확정한다.
- 모델 범위 확대는 별도 변경 승인 전까지 수행하지 않는다.

## PMD-002. 공식 근거와 공통 FAQ

- `WPU-IAC506 REV.00` 매뉴얼을 1차 공식 근거로 사용한다.
- 정수기 공통 FAQ는 WPU-IAC506 직접 적용이 확인되지 않은 조건부 보조 자료로 구분한다.
- FAQ가 매뉴얼과 충돌하거나 모델 적용 여부가 불분명하면 고객 안내 근거로 사용하지 않는다.
- 타 모델 전용 FAQ는 WPU-IAC506 고객 안내에서 제외한다.

## PMD-003. 방문관리·셀프관리 시연 규칙

- 방문관리·셀프관리 시연 규칙은 `team_designed` 상태로 유지한다.
- 데이터와 화면에서 공식 정책이 아닌 MVP 시연용 팀 설계 규칙임을 표시한다.
- 공식 기준과 `team_designed` 규칙을 혼합해 표현하지 않는다.

## PMD-004. 오류 코드 조사 범위

- 전체 공식 오류 코드 자료의 추가 조사는 현재 단계에서 진행하지 않는다.
- 공식 근거가 확보되지 않은 오류 코드는 임의로 해석하지 않는다.
- 고객이 입력한 미확인 오류 코드는 원문 그대로 저장하고 상담 연결 대상으로 처리한다.
- 대표 시나리오 또는 구현 과정에서 필요성이 확인되면 추가 조사를 다시 결정한다.

## PMD-005. 원본 문서 외부 공유

- 공식 매뉴얼과 FAQ 원본은 팀 내부 개발·검토용으로만 사용한다.
- 원본과 원본을 포함한 인계 ZIP은 공개 저장소에 업로드하지 않는다.
- 보고서와 시연 결과에는 문서명·공식 출처·버전·페이지 등 필요한 최소 근거만 표시한다.
- 원문 전체의 외부 배포는 별도 권한 확인 후 결정한다.

저장소 보호 규칙으로 `data/raw/`와 `data/handoff_packages/*.zip`을 `.gitignore`에 등록한다. 이 규칙은 공개 저장소 실수 업로드를 막기 위한 최소 조치이며 원본의 재배포 권한을 의미하지 않는다.

## PMD-006. 팀원 인계와 다음 단계

- 팀원 B·D·E 인계와 다음 구현 단계 진행을 승인한다.
- 팀원 B에게 RAG 문서·메타데이터·전처리 기준·대표 샘플을 전달한다.
- 팀원 D에게 데이터 구조·식별값·합성 시나리오·저장 기준을 전달한다.
- 팀원 E에게 화면 근거 표시·상태 안내·시연용 데이터를 전달한다.
- 온수 일반 점검과 히터 경고 청크 분리는 후속 전처리·안전성 보완 항목으로 관리한다.

## 별도 기술 협의로 남는 항목

다음 항목은 PM 정책 승인으로 자동 해결되지 않으므로 구현 전에 팀원 D·E와 데이터 계약을 확정해야 한다.

1. `risk_level`과 `current_use_guidance`의 독립 코드 체계
2. 상담·방문 여부에 따라 분기되는 문의 상태 전이
3. 합성 고객명·연락처·기사명 표시와 마스킹 방식
4. 공식 원문과 공식 근거 기반 가공 요약의 분류
5. `Document`·`RetrievalResult` 필드와 FAQ 콘텐츠 ID 저장 위치
6. 온수 일반 점검 청크와 히터 경고 청크의 분리 기준
